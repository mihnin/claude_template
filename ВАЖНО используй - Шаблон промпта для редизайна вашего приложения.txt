Шаблон промпта для запуска процесса редизайна сайта

Основная Задача:

    URL сайта для редизайна: [ВСТАВЬТЕ URL ВАШЕГО САЙТА ЗДЕСЬ]

    Главные цели редизайна: [ОПИШИТЕ, ЧТО ВЫ ХОТИТЕ УЛУЧШИТЬ, например: "обновить устаревший дизайн, улучшить мобильную адаптивность и повысить скорость загрузки, сохранив при этом весь текущий функционал"]

    Технологический стек (если известен): [УКАЖИТЕ, например: SharePoint, React, чистый JS и т.д.]

План Выполнения и Правила Взаимодействия:

    Рабочий процесс и специальные инструкции: Для выполнения этой задачи, пожалуйста, используй субагенты в следующей последовательности, применяя на каждом шаге указанные инструкции:

        Шаг 1: Анализ существующего сайта (site-redesign-analyst)

            Инструкция: Проведи полный аудит сайта по указанному URL. Задокументируй текущий дизайн, функционал и технический стек.

        Шаг 2: План редизайна (idea-validator)

            Инструкция: Задача этого шага — не создание новой идеи, а формирование плана редизайна для существующего сайта. Используй результаты анализа из Шага 1, чтобы предложить современное визуальное видение и UX-улучшения.

        Шаг 3: Анализ требований (requirements-analyst)

            Инструкция: Главный приоритет — сохранить 100% существующего функционала. Твоя задача — описать требования так, чтобы они отражали только обновление визуальной части и пользовательского интерфейса, не затрагивая бизнес-логику.

        Шаг 4: Новая дизайн-система (ui-architect)

            Инструкция: Создай современную и гибкую дизайн-систему. Она должна быть совместима с существующим технологическим стеком (например, [SharePoint/React]) и учитывать возможность прогрессивного внедрения.

        Шаг 5: Прогрессивное улучшение (component-builder)

            Инструкция: Твоя задача — не создавать компоненты с нуля, а "обернуть" существующие компоненты в новые стили из созданной дизайн-системы. Это должно быть сделано максимально аккуратно, чтобы не нарушить их работу.

        Шаг 6: Пропуск интеграции с API (api-integrator)

            Инструкция: Этот шаг пропускается. API уже интегрировано и не должно изменяться в процессе редизайна.

        Шаг 7: Оптимизация производительности (performance-optimizer)

            Инструкция: Проведи оптимизацию только фронтенд-ассетов: CSS, JS и изображений. Не изменяй существующую логику работы приложения.

        Шаг 8: Улучшение доступности (accessibility-tester)

            Инструкция: Проверь и улучши доступность существующих элементов интерфейса в соответствии с новыми стилями и стандартами WCAG.

    Протокол Взаимодействия (Очень важно):

        После завершения работы каждого агента, ты обязан остановиться.

        Сообщи мне, какой шаг выполнен и какие ключевые документы были созданы в папке docs/.

        Предложи мне внимательно изучить созданные документы или результаты работы.

        Затем задай мне прямой вопрос для продолжения: "Продолжаем?"

        Не переходи к следующему агенту, пока не получишь от меня явного подтверждения.
